# 项目构建 - 环境配置

这份文档只讲一件事：环境配置

---

## 一、本地运行需要准备什么

### 1. 安装 Python

建议使用：

- Python 3.12+（CI 当前测试 3.12 与 3.14）

当前项目依赖 Playwright 和 Pydantic，  
用较新的 Python 版本会更省心。

### 2. 创建虚拟环境

在项目根目录执行：

```powershell
python -m venv .venv
```

### 3. 激活虚拟环境

```powershell
.venv\Scripts\Activate.ps1
```

### 4. 安装依赖

```powershell
python -m pip install --upgrade pip
pip install -r requirements.txt
```

### 5. 安装 Playwright 浏览器

```powershell
playwright install chromium
```

## 二、配置文件怎么准备

先把：

```text
.env.example
```

复制成：

```text
.env
```

然后填写你自己的 Pixiv 账号密码。

当前主要配置项有：

- `PIXIV_USERNAME`
- `PIXIV_PASSWORD`
- `HEADLESS`
- `PROXY_SERVER`
- `PROXY_USERNAME`
- `PROXY_PASSWORD`
- `TIMEOUT`
- `SAVE_DEBUG_ARTIFACTS`
- `VERBOSE_DEBUG_OUTPUT`
- `DOWNLOAD_DIR`
- `STATE_FILE`
- `DB_PATH`
- `LOG_PATH`

### 推荐配置

本地首次运行时，建议：

```env
HEADLESS=false
```

这样浏览器会弹出来，  
如果 Pixiv 触发 `reCAPTCHA`，你还能手动完成验证。

### 调试推荐配置

如果只是日常下载，建议：

```env
SAVE_DEBUG_ARTIFACTS=false
VERBOSE_DEBUG_OUTPUT=false
```

如果你要排查“页面打开了但解析不对”或者“下载器没有拿到图”的问题，再临时打开：

```env
SAVE_DEBUG_ARTIFACTS=true
VERBOSE_DEBUG_OUTPUT=true
```

打开后可以同时看：

- 终端里的 `DEBUG` 日志
- `data/temp/html`
- `data/temp/json`

## 三、登录态文件是什么

项目会把登录态保存到：

```text
data/state/storage_state.json
```

这个文件里包含：

- cookies
- localStorage
- 其他登录相关状态

作用是：

- 下次启动时复用登录态
- 避免每次都重新登录

### 注意

这个文件是敏感文件，注意自己的隐私安全。
如果登录失败，当前主流程会直接退出，不会再继续误跑后续抓取任务。

## 四、第一次运行推荐顺序

### 方案 A：宿主机直接跑

```powershell
python main.py
```

第一次建议这样跑，因为：

- 可以看到浏览器
- 遇到验证码可以人工处理
- 更适合先把登录态建立起来

### 方案 B：Docker 跑

如果你已经在宿主机上生成过：

```text
data/state/storage_state.json
```

那么就可以用 Docker 复用它。

---

## 五、Docker 环境怎么准备

### 1. 安装 Docker Desktop

新机器如果要走 Docker，先装：

- Docker Desktop

### 2. 准备好这些文件

- 项目源码
- `.env`
- `data/state/storage_state.json`

### 3. 构建镜像

```powershell
docker compose build
```

### 4. 启动容器

```powershell
docker compose run --rm pixiv-crawl
```

当前容器会挂载：

- `./data`
- `./logs`

所以容器里产生的下载图片、数据库、登录态仍然会保存在宿主机。

---

## 六、Windows 和 Docker 的一个小差别

`requirements.txt` 里已经做了平台兼容：

- `win32_setctime` 只会在 Windows 安装
- Linux Docker 容器里不会再因为这个依赖报错

## 七、哪些文件不要上传

这些都不应该上传 GitHub：

- `.env`
- `data/state/`
- `data/images/`
- `data/exports/`
- `data/*.db`
- `data/temp/`
- `logs/`

---

## 八、环境是否正常的最小检查

如果想确认环境装好了，可以跑：

```powershell
python -m unittest discover -s tests -v
```

如果这些都能过，通常说明：

- Python 环境正常
- 依赖安装正常
- 项目代码至少能通过基础回归检查
- 登录、终端展示、解析器和下载链路都能做基础回归
