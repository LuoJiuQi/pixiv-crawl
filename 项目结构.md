## 目录职责

### `app/browser`

负责浏览器基础能力：

- 启动和关闭 Playwright
- 复用登录态
- 自动登录
- `reCAPTCHA` 人工补充验证

### `app/core`

负责统一配置：

- 账号密码
- 超时
- 下载目录
- 数据库路径
- 日志路径

### `app/crawler`

负责进入作品页并采集原始内容：

- 打开作品详情页
- 读取页面 HTML
- 保存调试样本

### `app/parser`

负责把 HTML 解析成结构化作品信息：

- 标题
- 作者
- 标签
- 页数
- 图片候选地址

### `app/downloader`

负责图片下载：

- 复用浏览器 cookies 下载
- 自动选择原图
- 多图作品自动全下
- 自动跳过已下载文件

### `app/db`

负责 SQLite 任务记录：

- 记录作品处理状态
- 查询历史记录
- 统计状态和错误类型
- 删除旧记录

### `app/services`

负责业务辅助服务：

- 失败原因分类
- 失败清单导出
- 归档记录导出

### `app/utils`

负责调试辅助工具：

- HTML / JSON 格式化

### `data/temp`

这是当前项目里非常重要的调试目录：

- `html/`：保存作品页源码样本
- `json/`：保存解析结果样本

这些文件既能用于手动排查问题，也能直接给测试复用。

## 已完成能力

当前已经完成：

- 自动登录与登录态复用
- 作品详情页抓取
- 作品信息解析
- 多图作品自动下载
- 批量输入作品 ID / 作品链接
- SQLite 任务记录
- 已完成任务跳过
- 失败任务重试
- 失败清单导出
- 旧记录归档与清理

当前还没正式做的部分主要有：

- 搜索页批量抓取
- 排行榜抓取
